## Draft for job #1
Hi,

This is very close to a workflow I would build as a structured sales automation system, not just a random set of Zaps.

I built a small demo showing the exact first layer: inbound lead → AI classification → CRM record → AI draft reply → follow-up task:
https://janisag07.github.io/upwork-job-demos/publish/ai-sales-pipeline-automation/?v=f57de67

For your phases, I would keep it clean:
- Phase 1: Gmail trigger, AI classification, Pipedrive/Close contact + task, reply draft, follow-up reminder
- Phase 2: reply detection, deal stage update, quote email template with mapped fields
- Phase 3: won deal sync to Airtable/Notion, invoice template, monthly summary export

For AI misclassification, I would not let the AI silently control the whole workflow. I’d use confidence scoring, fallback categories, and a review queue for low-confidence emails so the system stays reliable.

I have worked with CRM/workflow automation logic and can document the setup clearly so it is easy to edit after handoff.

Best,
Janis

Screening answers to include:
1. Similar automation: inbound lead/contact workflows with CRM routing, tags, follow-up tasks, and AI-generated response drafts.
2. Tools: Zapier/Make/n8n style workflows, OpenAI/Anthropic classification, CRM fields/stages, Airtable/Notion reporting.
3. Misclassification: confidence threshold, default fallback category, human review queue, logging and examples to improve prompts.
4. Pipedrive vs Close: Pipedrive is usually better for visual pipeline simplicity; Close is better if calling/emailing inside the CRM is central.
