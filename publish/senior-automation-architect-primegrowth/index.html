System architecture

Recommended split: entry channels capture context, n8n owns validation/routing, GHL owns CRM, workflows, pipeline and conversations.

Entry pointsMeta Ads, IG/FB DM, website forms, calculator, apply page, booking.
→
n8n intake layerWebhook per source, schema normalization, validation, duplicate lookup, routing.
→
GoHighLevelContact, opportunity, tags, custom fields, bot assignment, calendar, nurture.

ManyChat path

  • Trigger from IG/FB DM keyword or ad comment.
  • Collect name, phone/email, trade, city, urgency.
  • Send payload to n8n webhook with source metadata.

Vercel website path

  • React forms submit to environment-based endpoint.
  • UTM, page path and calculator results are included.
  • GitHub PR updates endpoint safely before Vercel deploy.

GHL workflow path

  • Contact create/update + opportunity creation.
  • Tags assign source, service type, intent and language.
  • Conversation AI or nurture workflow starts by route.

Duplicate prevention logic

The system should never blindly create a new contact. n8n should first search GHL by normalized phone, then email, then fallback match rules. If found, update the existing contact and append source history instead of overwriting important attribution.

CheckAction
Phone exact matchUpdate existing contact, add latest source tag, append note.
Email exact matchUpdate if phone missing or confirm as same lead.
Name + city + recent timestampFlag for review or soft-merge to avoid false positives.
Existing open opportunityDo not create another opportunity. Move/update stage if needed.
incoming = normalize(payload)
contact = ghl.find_by_phone(incoming.phone)
       or ghl.find_by_email(incoming.email)

if contact:
    ghl.update_contact(contact.id, incoming.fields)
    ghl.add_tags(contact.id, incoming.tags + ["source-history-updated"])
    ghl.add_note(contact.id, f"New touchpoint: {incoming.source}")
    opportunity = ghl.find_open_opportunity(contact.id)
    if not opportunity:
        ghl.create_opportunity(contact.id, pipeline_stage="New Lead")
else:
    contact = ghl.create_contact(incoming.fields)
    ghl.create_opportunity(contact.id, pipeline_stage="New Lead")

React / Vercel endpoint update

  1. Locate the form submit handler or API route.
  2. Move webhook URL into an environment variable like NEXT_PUBLIC_LEAD_WEBHOOK_URL or server-side secret.
  3. Create a GitHub branch, update code, test locally, commit with a clear message.
  4. Open PR or merge to main, let Vercel deploy automatically, verify the deployment URL.
  5. Submit test lead and confirm n8n execution + GHL contact update.
