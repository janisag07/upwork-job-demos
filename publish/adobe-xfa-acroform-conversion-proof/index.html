Exact delivery path

This is the workflow I would use for the real form: preserve layout first, then rebuild the field technology underneath it.

01

Audit the old PDF

Identify XFA/LiveCycle structure, page count, fonts, field locations, signatures, validation rules, and browser/automation failure points.

02

Recreate layout

Match the original form visually without redesigning it: spacing, borders, labels, page breaks and government-form hierarchy.

03

Build AcroForm fields

Create standard text/date/signature fields with stable field names for JSON, Python, Node or SaaS population.

04

Fill + flatten test

Generate a filled editable PDF from JSON and a flattened final PDF, then verify no XFA/LiveCycle remains.

Why this matches the job

No redesign

The priority is visual parity with the old government form, not a new layout.

No XFA

The master is normal AcroForm so it can work with web apps, AI tools, Python, Node and SaaS workflows.

Clear handoff

Deliverables include master PDF, editable/source reference, field list, JSON sample, filled PDF and flattened final.

Verification

Before delivery I would run field-count, no-XFA and flatten checks so the result is testable, not subjective.
