What I would build first

A tight proof workflow: extract the storyboard into shot cards, define Kling prompt templates, generate 3-5 test clips, track failures/retries, then assemble a review cut with notes for voiceover, sound and pacing.

Why this helps

The risk in AI narrative video is not generation itself — it is consistency, shot control and knowing which failures to regenerate. This gives the project a repeatable production system instead of one-off prompting.

This is a lightweight demo interface using fictional scene data. Final implementation would adapt to your actual zip/storyboard/materials and rights constraints.
