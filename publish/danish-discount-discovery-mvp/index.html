Proof-of-approach for the Upwork brief

Danish Discount Discovery MVP

Search a webshop like Zalando, Power, or IKEA → normalize offers from Danish discount programs → show trusted savings sources with a lightweight admin panel and scraper status board.

Frontend

Mobile search + autocomplete

Fast search experience with store cards, offer badges, and clear reveal flow for discount sources.

Backend

Flexible offer/admin schema

Admin panel edits shops, programs, %, fixed amount, and custom benefit types without schema rewrites.

Data

Scraping normalization map

Source adapters write into one normalized offer table with confidence and review flags.

Build path

Start with Figma pages, implement mobile-first search/autocomplete, store pages, program pages, ad reveal, admin CRUD, deployment, and clear documentation.

Data model

Shops, programs, offers, offer type, value, validity, source URL, login-required flag, confidence status, and last-scraped timestamp.

Scraper QA

11 initial sources separated into public, login-required, manual-review, and failed states so the MVP ships with realistic data quality checks.

Milestone 1

Clickable search/store/admin MVP with seeded Danish sample data, one scraper adapter pattern, and deployment checklist before expanding sources.

Fictional data only. Built quickly to show product thinking, architecture, and implementation approach for the exact Upwork MVP scope.
